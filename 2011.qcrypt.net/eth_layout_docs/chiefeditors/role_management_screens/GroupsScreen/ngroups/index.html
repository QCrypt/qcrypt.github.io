LDAP-Gruppen einrichten

Authorisierngs-Gruppen (auch nethz-Gruppen) können für LDAP oder auch Exchange erstellt werden. Sie werden deshalb nicht lokal in Ihrer WCMS-Site erstellt, sondern mit einem nethz-Admin-Programm. LDAP-Gruppen sind ETH-weit in WCMS-Sites sichtbar.

Viele Authorisierngs-Gruppen sind vordefiniert, z.B. ganze Leitzahlen umfassende Gruppen oder Domänen siehe Bsp.


Zur Erstellung von Authorisierngs-Gruppen wählen Sie in Ihrem Browser die URL
https://idn.ethz.ch/cgi-bin/admin_tool/main.cgi. Loggen Sie sich mit Ihren nethz-Angaben ein:


und drücken Sie die Schaltfläche „login“.

Haben Sie genügend Rechte, um Authorisierngs-Gruppen zu erstellen, so sehen Sie im folgenden Fenster die Angabe „Administrator beim Nethz Administrations Tool.“


Wählen Sie „Admin Tasks“ im Menü und anschliessend Gruppen verwalten:


Klicken Sie hier den Namen einer bestehenden Gruppe an, um sie zu verändern. Soll eine neue Gruppe erstellt werden, so wählen Sie


Der Gruppenname wird nach dem Muster „ID-TIM-Kurse“
1.    Feld: ID (wird Ihrem Konto entnommen, entspricht Departementskürzel)
2.    Feld: TIM (Institut oder Gruppennamen)
3.    Feld: Kurse (Gruppenname oder Thema)

eingegeben.
Für Silva muss „Authorisierungsgruppe (LDAPS) ausgewählt werden.

Speichern Sie die neue Gruppe. Sie wird nun unter den Gruppen alphabetisch aufgeführt. Klicken Sie deren Namen an, um Benutzer in die neue Gruppe aufzunehmen.

Fügen Sie wie folgt Einzelpersonen hinzu:


Schreiben Sie im folgenden Formular Suchangaben und wählen Sie dann „User suchen und anzeigen“.  Angaben brauchen nur in 1 Feld eingegeben werden.


Nun erhalten Sie eine Liste von gefundenen Einträgen aus der Datenbank zurück.

Treffen Sie eine Auswahl einzelner oder wählen Sie alle mittels „alle wählen“ und klicken Sie auf die Schaltfläche „User hinzufügen“.


Die aufgenommen Personen werden anschliessend im Gruppenfenster aufgeführt.


Nachts werden die LDAP-Gruppen exportiert, d.h. ab diesem Zeitpunkt erst können diese Gruppen in Silva-Sites eingesetzt werden. Sie sind dann für Chiefeditoren in Silva sichtbar im „access-Reiter“ unter „existing groups“:

Solchen Personengruppen können Sie nun in Ihrer Site Rechte zuweisen.
